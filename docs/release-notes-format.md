# Release Notes Format

Write release notes on GitHub using these sections:

```markdown
### New
- Feature description here
- Another new feature

### Improved
- Something that got better
- Another improvement

### Fixed
- Bug that was squashed
```

Only include sections that apply — skip any section with nothing to report.

## Example (v2.0.1)

```markdown
### Improved
- Account card layout: tighter spacing, smaller In/Out/Net amounts
- Dimmed input placeholder text for better contrast
- Updated Stats tab icon
- New accounts now added to the bottom of the list
```

## Rules

- One bullet per change
- Plain language, user-facing — not technical commit messages
- `### New` → net-new features
- `### Improved` → enhancements to existing features, UI polish, performance
- `### Fixed` → bug fixes
